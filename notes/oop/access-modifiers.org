#+TITLE: The organization of classes in Java

** Grouping

When you create any *top-level* class, it will be created inside the *default package*

Unless you specify a package name, using the /package/ statmenet


*** Logical/Physical Grouping

Packages is considered a *logical grouping* because it groups classes under a specific category

and it also considered as *physical grouping*, because each package is a path of folders and files in the project


** Java's Access Modifers


*** public (world-readable)

Accessible *by anyone at everywhere*

/Inheritance/: cross-package-inheritance

---------------------------------------------------------------------------

*** none (no access modifier)

Accessible only to classes *within the same package*

/Inheritance/:

- package-private-inheritance
- *sub classes* from other packages cannot inherit *package-private* members


---------------------------------------------------------------------------

*** private

Accessible *only inside the class*, even instances of that class cannot access it

/Inheritance/: cannot be inherited


---------------------------------------------------------------------------

*** protected (package-private with cross-package inheritance)

The *protected* access modifier has two faces:

- *within the same package*:
  - it behaves like *public*, so there's no difference between it and *none (package-private)*

- *within other packages*:
  - it behaves like a *private inheritance*, meaning it behaves like a private member which is accessible only inside the subclass, but at the same time it's inheritable and *SubClasses within other packages can inherit it but they will inherit it as a private member*.

  - *protected fields* are accissbile from *subclasses* only outside the package.

/Inheritance/: corss-package-inheritance for *sub classes*
